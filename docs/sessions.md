# Sessions and Artifacts

Sessions are the history of agent work. Artifacts are what the work produced.

A useful session view should show:

- what started the run
- which room it happened in
- which agents were involved
- which runtime powered it
- major handoffs
- stalls and interventions
- produced artifacts
- replay or timeline state

## Why This Matters

Agent teams can run longer than a human can comfortably watch. Agentarium needs to make that work reviewable after the fact.

The operator should be able to answer:

1. What happened?
2. What changed?
3. What did the agents produce?
4. Was there an escalation?
5. Can we replay or fork from a useful moment?

## Public Proof

Some sessions may produce decisions, evidence, or verdicts that deserve public proof. Those flows belong in the Council and Base integration layer.
