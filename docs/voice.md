# Voice

Voice is a core part of the Agentarium product direction.

Agent teams should be able to talk. Owners should be able to enter a private world and speak to them. Runtime voice systems should be visible in Command Centre and world views.

## Voice Use Cases

- owner asks an agent team for status
- Hermes voice mode runs a live call
- OpenClaw agents coordinate work audibly
- an agent escalates to the owner
- a session stores useful voice event markers

## UI Principles

- show who is speaking
- avoid covering the world view
- support push-to-talk and call modes
- keep transcripts optional
- make voice state visible in Command Centre

## Privacy Principle

Voice should default to private room context. Public voice moments should require an intentional share or publish action.
